Overall Site Health

2.9/5 ‚Äî Adequate Performance with Optimization Needed

Desktop experience is exceptional with Lighthouse scores of 95-98 and sub-second load times, demonstrating solid technical foundations. However, mobile users face a significantly different reality: 4-4.5 second LCP times and over 1 second of blocking time create friction during the critical first impression. With mobile traffic typically representing 60-70% of fashion eCommerce sessions, this performance gap means the majority of visitors experience delayed product visibility and reduced interactivity. The homepage's 0.389 CLS adds visual instability during load, potentially causing mis-taps on navigation or product cards. These mobile bottlenecks directly impact bounce rates and conversion, as studies show each additional second of load time can reduce conversions by 7-10%. The good news: excellent desktop performance proves the infrastructure is sound, making mobile optimizations highly achievable through targeted image preloading, script deferral, and layout stability improvements.

Performance Risks (Mobile Network Constraints)

  • Mobile LCP Delays: 3.8-4.5s affecting users on slower connections
  • Third-Party Scripts: 14 external domains, 380KB blocking JS
  • Homepage CLS: 0.389 causing visual instability during load
  • Total Blocking Time: 880-1080ms on mobile throttled conditions

Strengths (Foundation Solid)

  • Excellent Desktop Performance: 95-98 Lighthouse scores
  • Strong Accessibility: 77-79% scores across pages
  • Low CLS on PLP/PDP: 0.105-0.266 indicating stable layouts
  • Modern Tech Stack: Shopify platform with optimization potential

Optimization Opportunities

  • Defer Third-Party Scripts: Load after core content (380KB reduction)
  • Image Optimization: Preload LCP images, optimize hero imagery
  • CLS Improvements: Reserve space for dynamic content on homepage
  • Resource Loading: Eliminate render-blocking resources

Mobile Performance Priority

Desktop experience is excellent, but mobile users face significant performance challenges. With mobile LCP times ranging 3.8-4.5 seconds and TBT over 1 second, users on slower networks experience delayed interactivity and content visibility. Third-party scripts from 14 external domains consume 380KB of render-blocking JavaScript. Prioritize mobile-first optimizations and third-party script management to close the performance gap.

Top 5 Optimization Priorities

  1. Defer Third-Party Scripts: Move GA4, FB Pixel, Klaviyo to post-interaction (reduces TBT by 400ms)
  2. Preload LCP Images: Add fetchpriority="high" to hero images, remove lazy loading from above-fold content
  3. Fix Homepage CLS: Reserve space for dynamic elements, set explicit dimensions on images
  4. Optimize Critical CSS: Inline critical styles, defer non-essential CSS
  5. Improve Color Contrast: Ensure 4.5:1 ratio on all text and CTAs (Accessibility)

Key Performance Bottlenecks

  • Mobile LCP Delays: Hero images and above-fold content loading slowly on constrained networks (3.8-4.5s)
  • Homepage Layout Shift: CLS 0.389 indicating dynamic content causing reflows during load
  • Render-Blocking Resources: CSS and JavaScript delaying First Contentful Paint by 2.4-2.6s on mobile
  • Heavy Main Thread Work: Total Blocking Time 880-1080ms preventing user interaction
  • Third-Party Impact: Analytics, marketing pixels, and app scripts adding overhead

‚ö° Performance Quick Wins (Highest Impact)

  1. Preload LCP Images:
    Add to theme head: <link rel="preload" as="image" href="hero-image.jpg" fetchpriority="high">
    Remove loading="lazy" from above-fold images
    Expected Impact: Reduce LCP by 1-2 seconds on mobile
  2. Fix Homepage CLS (0.389):
    Set explicit width and height on all images
    Reserve space for dynamic content with CSS min-height
    Expected Impact: Reduce CLS to <0.1
  3. Defer Non-Critical JavaScript:
    Add defer or async to analytics/marketing scripts
    Move third-party pixels to load after user interaction
    Expected Impact: Reduce TBT by 300-500ms
  4. Optimize Critical CSS:
    Inline critical styles for above-fold content
    Load remaining CSS asynchronously
    Expected Impact: Reduce FCP by 500-800ms
  5. Implement Resource Hints:
    <link rel="preconnect" href="//cdn.shopify.com">
    <link rel="dns-prefetch" href="//analytics.example.com">
    Expected Impact: Faster third-party resource loading

‚ö†Ô∏è Technical Stack Issues

  • Third-Party Script Cascade: 14 external domains loading synchronously, blocking main thread for 820ms total
  • Analytics Redundancy: Multiple tracking providers (GA4, FB, TikTok, Klaviyo) with overlapping functionality
  • Global Script Loading: Reviews widget and app scripts loading on all pages instead of where needed
  • Missing Script Management: No centralized tag management system for controlling third-party load order
  • Monitoring Gap: No real-user monitoring (RUM) to track production performance impact

‚öôÔ∏è Site Operations & Script Management Recommendations

  1. Implement Script Loading Strategy:
    Phase 1 - Immediate loads: Shopify Analytics only (required for checkout)
    Phase 2 - After interaction: GA4, FB Pixel, TikTok (use facade pattern)
    Phase 3 - After 3s: Hotjar, Klaviyo
    // Facade loading example window.addEventListener('scroll', () => { loadScript('https://www.googletagmanager.com/gtag/js'); }, {once: true});
    Impact: Reduce initial TBT by 400-500ms
  2. Consolidate Analytics Providers:
    Audit overlap between GA4, FB, TikTok, Klaviyo tracking
    Consider using GA4 as primary with server-side conversions to FB/TikTok
    Impact: Reduce script count from 14 to 8-10 domains
  3. Scope App Loading:
    Reviews widget: Load only on product pages
    Klaviyo forms: Load only when email capture is visible
    Use Shopify's app block controls to limit scope
    Impact: Save 100KB+ on non-product pages
  4. Implement Tag Management:
    Deploy Google Tag Manager or Segment for centralized control
    Set up triggers for conditional script loading
    Create firing rules based on user engagement
    Impact: Single point of control for all third-party scripts
  5. Set Up Performance Monitoring:
    Implement RUM solution (SpeedCurve, DataDog)
    Track Core Web Vitals by script provider
    Set alerts for performance regression
    Create dashboard showing script impact on conversion
    Impact: Data-driven decisions on script ROI vs performance cost

üõ°Ô∏è Third-Party Script Management Best Practices

Implementation Strategy:

  1. Audit Current Scripts: Document what each script does and its business value
  2. Create Loading Priority Matrix: Critical (checkout) ‚Üí Important (analytics) ‚Üí Nice-to-have (heatmaps)

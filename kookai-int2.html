Multi-Market Performance Overview

2.8/5 ‚Äî JavaScript Execution is Primary Bottleneck

GTmetrix testing across AU, UK, and US stores reveals consistent performance patterns: excellent LCP (630-721ms) but critically high Total Blocking Time (1.1-1.7s) driven by 3-4 seconds of JavaScript execution across all markets. The UK store faces an additional challenge with a 20.8MB homepage video inflating page weight to 25.8MB. These issues will compound during international consolidation regardless of architecture choice.

Primary Performance Bottlenecks

  • JavaScript Execution (3.0-3.8s): Main thread blocked by heavy script bundles, preventing user interaction
  • UK Homepage Video (20.8MB): Single video file causing 5x page weight vs other stores
  • Third-Party Script Blocking: 1.0-1.2s main thread blocked by external scripts (Klaviyo, GTM, Facebook)
  • Long Tasks (17-20 per page): Tasks exceeding 50ms threshold causing UI jank
  • CSS @import (3 resources): Render-blocking CSS loading pattern across all stores
  • HTTP/2 Gaps: 16-19 requests not served via HTTP/2

‚ö° Performance Quick Wins

  1. Defer Third-Party Scripts:
    Load Klaviyo, Clarity, Facebook Pixel, Tolstoy after user interaction
    Impact: Reduce TBT by 500-700ms
  2. Fix UK Homepage Video:
    Compress to <5MB, convert to WebM, or implement lazy loading
    Impact: Reduce UK page weight by 80%
  3. Code-Split Theme JavaScript:
    Load widget JS only on pages where needed (Tolstoy, Foursixty, KiwiSizing)
    Impact: Reduce initial JS payload by 30-40%
  4. Replace CSS @import:
    Use <link> tags for TypeKit fonts to enable parallel loading
    Impact: Faster FCP across all stores
  5. Unused JavaScript Cleanup:
    735-790KB of unused JS detected across stores
    Impact: Faster parsing and execution

Overall Site Health

2.9/5 ‚Äî Adequate Performance with Optimization Needed

Desktop experience is exceptional with Lighthouse scores of 95-98 and sub-second load times. However, mobile users face 4-4.5 second LCP times and over 1 second of blocking time. With mobile traffic typically representing 60-70% of fashion eCommerce sessions, this performance gap requires attention before international expansion.

üåè International Expansion Context

This assessment feeds into Kookai's international consolidation strategy. Two architecture options are being evaluated: Option 1 (Three Regional Stores) and Option 2 (Single Global Store). Performance foundations must be optimized before either approach to ensure consistent experience across AU, NZ, UK, EU, US, and CH markets. See the International Readiness tab for detailed analysis.

Performance Risks

  • Mobile LCP Delays: 3.8-4.5s affecting users on slower connections
  • Homepage CLS: 0.389 causing visual instability during load
  • Total Blocking Time: 880-1080ms on mobile throttled conditions
  • International CDN: Untested performance from UK/EU/US origins

Strengths

  • Excellent Desktop Performance: 95-98 Lighthouse scores
  • Strong Accessibility: 77-79% scores across pages
  • Low CLS on PLP/PDP: 0.105-0.266 indicating stable layouts
  • Shopify Platform: Native multi-currency/multi-market support

International Readiness

  • Multi-Currency: Shopify Markets supports local pricing
  • Geo-Detection: Requires implementation for market routing
  • App Compatibility: Audit needed for cross-market licensing
  • Content Localization: Theme supports translation apps

Top 5 Pre-International Priorities

  1. Preload LCP Images: Add fetchpriority="high" to hero images, remove lazy loading from above-fold content
  2. Fix Homepage CLS: Reserve space for dynamic elements, set explicit dimensions on images
  3. Defer Third-Party Scripts: Load analytics and marketing pixels after core content
  4. Implement Global CDN Testing: Benchmark performance from UK, EU, and US locations
  5. Audit App Stack: Identify apps requiring per-store licensing vs. global compatibility

‚ö†Ô∏è International Performance Consideration

Origin Server Location: Current metrics tested from AU. For UK/EU/US markets, expect additional latency of 150-300ms TTFB due to geographic distance unless Shopify's global CDN edge caching is optimized. Recommend running Lighthouse tests from target market locations before international launch.

Key Performance Bottlenecks

  • Mobile LCP Delays: Hero images and above-fold content loading slowly on constrained networks (3.8-4.5s)
  • Homepage Layout Shift: CLS 0.389 indicating dynamic content causing reflows during load
  • Render-Blocking Resources: CSS and JavaScript delaying First Contentful Paint by 2.4-2.6s on mobile
  • Heavy Main Thread Work: Total Blocking Time 880-1080ms preventing user interaction
  • Third-Party Impact: Analytics, marketing pixels, and app scripts adding overhead

‚ö° Performance Quick Wins

  1. Preload LCP Images: Add <link rel="preload" as="image" href="hero.jpg" fetchpriority="high">
  2. Fix Homepage CLS (0.389): Set explicit width/height on images, reserve space for dynamic content
  3. Defer Non-Critical JavaScript: Add defer/async to analytics and marketing scripts
  4. Optimize Critical CSS: Inline critical styles, load remaining CSS asynchronously
  5. Implement Resource Hints: Add preconnect to CDN and third-party domains

üè™ Current Multi-Store Architecture

Kookai currently operates 6 separate Shopify stores across different markets. This architecture is being evaluated for consolidation into either 3 regional hubs (Option 1) or a single global store (Option 2). Each approach has different implications for theme management, app licensing, and operational overhead.

‚ö†Ô∏è Technical Stack Considerations

  • Third-Party Scripts: Multiple analytics and marketing pixels adding overhead
  • App Dependencies: Current apps may require per-store licensing in multi-store setup
  • Theme Synchronization: 6 stores means 6x theme maintenance effort
  • Content Management: Product data, translations, and pricing across multiple stores
  • Monitoring Fragmentation: Analytics and RUM data split across stores

‚öôÔ∏è Site Operations Recommendations

  1. Audit App Licensing: Document which apps support multi-store vs. require separate licenses
  2. Implement Centralized Monitoring: Use RUM solution that aggregates across all stores
  3. Theme Version Control: Establish Git-based theme management for consistent deployments
  4. Create Performance Budget: Set targets: Mobile LCP <2.5s, CLS <0.1, TBT <300ms
  5. Document Rollback Procedures: Per-store and global rollback processes

üõ°Ô∏è Pre-Consolidation Checklist

  1. Backup All Themes: Export current themes from all 6 stores
  2. Export Customer Data: Ensure customer records are portable
  3. Document URL Structures: Plan 301 redirects for any URL changes
  4. Audit SEO Rankings: Baseline organic performance per market
  5. Test Payment Gateways: Verify gateway compatibility for each currency
